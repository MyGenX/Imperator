# Install Security

Imperator's installer scripts are thin wrappers around the shared Python
installer in `cli/imperator/installer.py`. They download or update this
repository, then install the Python CLI in editable mode. They do not modify your
projects until you later run `imperator init` inside a project directory.

## macOS, Linux, and WSL

`install.sh`:

1. Finds `python3` or `python`.
2. If run from a checkout, executes `cli/imperator/installer.py`.
3. If run through `curl | bash`, downloads the Python installer from
   `IMPERATOR_INSTALLER_URL` or the default raw GitHub URL.
4. Forwards all flags to the Python installer.

The Python installer then:

1. Reads `IMPERATOR_REPO`, defaulting to `https://github.com/MyGenX/Imperator`.
2. Reads `IMPERATOR_DIR`, defaulting to `$HOME/.imperator`, unless `--dir` is set.
3. Checks that `git` is available.
4. If the install directory is an Imperator git checkout, updates it.
5. Otherwise, clones `IMPERATOR_REPO` into the install directory.
6. Runs `python -m pip install -e "$IMPERATOR_DIR/cli"`.

## Windows PowerShell

`install.ps1`:

1. Finds `python` or `python3`.
2. If run from a checkout, executes `cli\imperator\installer.py`.
3. If run through `irm | iex`, downloads the Python installer from
   `IMPERATOR_INSTALLER_URL` or the default raw GitHub URL.
4. Forwards all flags to the Python installer.

PowerShell-style aliases such as `-DryRun`, `-Dir`, `-NoColor`, and
`-NonInteractive` are accepted by the Python installer.

## Environment Variables

| Variable | Meaning | Default |
|---|---|---|
| `IMPERATOR_DIR` | Local checkout path used by the installer and by the CLI when locating rules. | `$HOME/.imperator` on Unix-like systems, `$HOME\.imperator` on Windows |
| `IMPERATOR_REPO` | Git repository cloned or pulled by the installer. | `https://github.com/MyGenX/Imperator` |
| `IMPERATOR_INSTALLER_URL` | Raw URL for the Python installer used by remote wrapper installs. | `https://raw.githubusercontent.com/MyGenX/Imperator/main/cli/imperator/installer.py` |

## Installer Flags

| Flag | Meaning |
|---|---|
| `--dry-run` | Print clone, update, pip, or uninstall actions without changing install state. |
| `--force` | Force-refresh a recognized Imperator checkout before reinstalling. |
| `--uninstall` | Uninstall the Python package and remove the recognized Imperator checkout. |
| `--dir <path>` | Override `IMPERATOR_DIR`. |
| `--no-color` | Disable colored installer output. |
| `--non-interactive` | Do not wait for prompts. The current installer has no required prompts; unsafe states fail instead. |

Examples:

```bash
./install.sh --dry-run --dir ~/.imperator --no-color
./install.sh --force
./install.sh --uninstall
```

```powershell
.\install.ps1 -DryRun -Dir "$HOME\.imperator" -NoColor
.\install.ps1 -Force
.\install.ps1 -Uninstall
```

## Requirements

- Git
- Python 3.8+
- `pip`

The CLI package declares one Python dependency: `Jinja2>=3.0`.

## Telemetry

Imperator does not include telemetry code. The installer does not phone home
beyond the explicit Git clone or pull and the pip install command you run.

## Manual Install

For the most reviewable install path, clone and inspect the repository first:

```bash
git clone https://github.com/MyGenX/Imperator ~/.imperator
cd ~/.imperator
pip install -e cli
```

Then initialize a project from that installed CLI:

```bash
cd /path/to/your/project
imperator init
```

`imperator init` writes generated agent files and `.imperator.json` in the current
project. Review and commit those files if they match your team's policy.

## Lifecycle Commands

Use `imperator doctor` inside a project to verify the CLI, rule repository,
`.imperator.json`, generated output freshness, and detected agent files.

Use `imperator clean` to remove Imperator-generated agent files. Cleanup removes
only files with `imperator:begin generated` / `imperator:end generated` markers
or legacy `Generated by Imperator` comments. User-authored files in generated
directories are skipped and reported.
